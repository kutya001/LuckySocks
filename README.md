# SOCKS.PRO — Система управления и оцифровки трикотажного производства (LuckySocks)

**SOCKS.PRO** — это специализированная ERP/MES-система реального времени для оцифровки, учета и планирования на чулочно-носочной фабрике. Решение охватывает сквозную цепочку от приема заказа до отгрузки готовой продукции клиенту, обеспечивая жесткий контроль технологических остатков, брака, рабочих смен и окупаемости оборудования.

---

## 🏗️ Структура меню и функциональные модули

Интерфейс системы разделен на логические блоки, оптимизированные под роли пользователей (руководство, менеджеры продаж, технологи, бригадиры цехов):

### 1. Главная
* **Панель управления (Dashboard)**:
  - Сводные KPI за текущий месяц (активные заказы, выпуски по цехам вязания, швейки, упаковки).
  - Интерактивный статус вязальных линий (список работающих станков с цветовой индикацией).
  - Живая лента активности (автоматический аудит последних технологических операций).
* **Мониторинг этапов**:
  - **Выполнение заказов**: Информационный дашборд по каждому клиенту с раскладкой по стадиям (`Заказано` ➔ `Запланировано` ➔ `Связано` ➔ `Прошито` ➔ `Упаковано`) и цветовой шкалой прогресса.
  - **Производственно мощностной план-Факт**: Сводный интерактивный отчет для сравнения запланированных мощностей фабрики с фактической выработкой за выбранный месяц.

### 2. Продажа
* **Заказы покупателей**: Оформление контрактов в мультивалютном режиме (USD, EUR, RUB, KGS). Поддержка загрузки официальных курсов валют Национального Банка.
* **Реализация продукции**: Списание готовой продукции со склада. Поддерживает автозаполнение по открытым заказам клиентов и прямые продажи со свободных остатков.

### 3. Планирование
* **Производственно мощностной план (ПМП)**:
  - Месячное планирование ресурсов фабрики с ручным вводом месяца в формате `ГГГГ-ММ`.
  - Раздельные таблицы планирования по этапам: **Вязальный**, **Прошив**, **Упаковка**.
  - Наследование и синхронизация рабочих часов: изменение общей нормы в шапке автоматически обновляет рабочие часы во всех строках.
* **Планирование и запуск**: Распределение заказов клиентов в производство, генерация уникальных плановых номеров (номеров партий) для отслеживания.

### 4. Разработка
* **Спецификации**: Создание технологических карт на носки (норма расхода сырья, время прошива в секундах, рисунки, базовые характеристики).
* **Справочники**: Централизованное ведение НСИ (Номенклатура, Характеристики, Контрагенты, Договоры, Производственные линии, Сотрудники, Курсы валют).

### 5. Производство
* **Вязальный**: Регистрация выработки вязальщиков. Фиксация номеров вязальных станков, операторов смены и сданных штук полуфабриката.
* **Прошив**: Сборка мыска швеями. Учет выработки сотрудников в парах готовой продукции.
* **Упаковка**: Сдача готовых носков на склад. Детальный учет по сортам (1-й сорт, 2-й сорт, 3-й сорт) с блокировкой сдачи свыше прошитого объема.

## ⚙️ Бизнес-логика и жесткие валидации

Главной ценностью SOCKS.PRO является встроенный механизм пресечения ошибок персонала и защита от потерь:

1. **Живой контроль остатков в табличных частях**:
   При добавлении строк в любые производственные документы система на лету рассчитывает и выводит 3 служебные колонки:
   * *Всего на пред. этапе* — сколько материала дошло до текущего шага.
   * *Уже обработано* — сколько из этого объема уже проведено другими документами.
   * *Осталось обработать* — лимит, который разрешено ввести в текущей строке.
2. **Блокировка перепроизводства и отрицательных остатков**:
   * Вязальщик не может сдать заготовок больше, чем запланировано.
   * Швея не может сшить носков больше, чем связано.
   * Упаковщик не может упаковать больше, чем прошито.
   * Реализация не позволяет отгрузить товар в минус при отсутствии на складе готовой продукции.
3. **Автоматический расчет мощностей в ПМП**:
   * $\text{Плановый объем сек} = \text{Оборудование} \times \text{Рабочие дни} \times \text{Часы работы} \times 3600 \times (\text{Коэффициент выработки} / 100)$
   * $\text{Плановый объем пар/шт} = \text{Плановый объем сек} / \text{Норма сек}$

## 📱 Мобильная адаптация (Mobile Responsive UX)

Интерфейс системы полностью оптимизирован для работы на смартфонах и планшетах:

* **Нижняя навигация**: Быстрый доступ к «Панели управления» и «Мониторингу этапов» с нижнего бара. Дополнительные разделы доступны в выдвижной шторке (Drawer), которая динамически клонирует структуру десктопного меню.
* **Карточный вид реестров**: Широкие десктопные таблицы автоматически трансформируются в аккуратные карточки (`.mobile-card`). Пользователь может гибко настраивать видимость полей на карточках через меню «Колонки».
* **Плавающая кнопка FAB (`+`)**: Расположена в правом нижнем углу под большой палец правой руки для быстрого добавления новых документов на ходу.
* **Адаптивные формы**: Поля ввода выстраиваются в одну колонку. Табличные части спецификаций документов упакованы в контейнеры с плавной горизонтальной прокруткой, защищая формы от обрезки и обеспечивая удобство ввода.

## 🛠️ Технологический стек и архитектура

* **Frontend**: Чистый HTML5, Vanilla CSS3 (современный адаптивный дизайн, HSL-палитра, переменные, анимации) и современный асинхронный JavaScript (ES6+).
* **База данных**: Локальное хранилище браузера (`LocalStorage`) с полной персистентностью состояния. Предусмотрена кнопка сброса и предзаполнения демонстрационными данными для тестирования.
* **Ядро системы**:
  - `BaseEntityComponent`: Базовый класс для справочников и простых реестров. Отвечает за пагинацию (5, 10, 20, 50, 100 строк), полнотекстовый поиск, фильтрацию по колонкам и экспорт данных в CSV.
  - `BaseDocument`: Специализированное ядро для многострочных документов с поддержкой статусов проведения, модальных форм редактирования и печатных форм.

## 🚀 Запуск проекта

1. Склонируйте репозиторий с проектом.
2. Дважды кликните по файлу `run.bat` (или просто откройте `index.html` в любом современном браузере).
3. Для тестирования системы нажмите на круглую кнопку сброса данных в правом верхнем углу (это заполнит базу демо-заказами, планами и выработкой).
